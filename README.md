Treasure Quest: A Hybrid Board Game for Python Control Flow Learning

Project Overview
This project is an Educational Hybrid Game designed to assess and enhance users' understanding of Python Control Flow such as if-statements, loops and so on through an interactive board game experience. It bridges physical gameplay with a digital platform to provide real-time feedback and progress tracking.

Technical Implementation
This project demonstrates strong experience in building secure, data-driven web applications with a focus on hybrid interactivity:
Full-Stack Development: Built using Node.js and Express.js for robust backend logic with Pug and client-side JavaScript for dynamic frontend rendering and CSS for a responsive user interface.
Database Management: Integrated a MySQL database with a complex schema to manage game sessions, player inventory including coins, cards and treasures and question attempts.
Secure Authentication: Implemented a login system to preserve user data and track game history across sessions.
Interactive Hybrid Mechanics: Developed logic for QR Code integration, allowing physical board movements to trigger digital events such as quizzes, shop access and treasure verification.

Key Game Features
Dual Roles: Specialized interfaces for Players (gameplay, shop, inventory) and Manager Hosts (session control, monitoring, code generation).
Dynamic Shop System: Players use earned coins to purchase digital Clue Cards to assist in finding treasures.
Dual Difficulty Modes: Offers Normal and Challenge modes, providing varying levels of rewards based on question difficulty.
Real-Time Monitoring: The Manager Host can track player movements and item acquisition in real-time to ensure fair play.

System Architecture
The system is built on a Client-Server Architecture:
ERD & Data Modeling: A structured database ensures data integrity for player rewards and session states.
Responsive Design: Optimized layouts for both Mobile, Tablet and Laptop users which ensuring accessibility during physical gameplay.
